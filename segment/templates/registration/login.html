Why this name?

Honeybox.ml is a wordgame combining the term "honeypot project used by Andrew Ng in a video talking about what to do next in a ML project" and the bounding boxes that a user does to capture the segments :)

What is HoneyBox.ml?

We can segment images by rectangles and tag this segments with one or more tags. I think that this is applicable to a variety of problems such as object detection( I am using it for this), face detection, face recognition, pedestrian detection, etc... not only for text. I hope I will be able to release some instruccions on the following days. For now, just a brieve description of functionalities:

1) The system is organized in training sets composed by one or more images. Each image can have one or more segments which can be tagged with one or more tags. The tags used with every segment are shared with every segment and every image that belongs to the same training set.

2) Once a training set( a set of images composed by segments) is tagged, the user can download the training set in a variety of formats(color,grayscale,edge detection) all in matlab or zip format. The matlab files are organized as follows:
  • X is a matrix with a row for every image.
  • labels is an ordered list of strings with the tag names corresponding to variable Y.
  • A binary matrix where the ith row indicates whether the ith image of X is tagged with tag j or not.
The zip format is very simple: a folder for every tag wich contains the images tagged by this tag. 3) Another interesting functionality allows you to generate a set of images for a second training set starting from a set of tags of a training set or image. The first applicability of this use case is for computer vision pipelines such as the described on the photo ocr videos but I think that could be useful on other problem domains.
